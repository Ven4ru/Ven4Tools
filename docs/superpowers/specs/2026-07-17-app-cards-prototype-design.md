# Прототип: карточка приложения в каталоге (в стиле Steam)

Статус: прототип, изолированная копия репозитория (`Ven4Tools-cards-prototype`, локально, без remote). Не часть основного релизного цикла Ven4Tools — цель - проверить сам концепт карточки, а не выпустить фичу.

## Идея

Сейчас каждая строка каталога — это чекбокс с именем и рядом кнопок (избранное/версия/Play/альтернатива/удалить-из-списка). Пользователь хочет, кликнув по названию приложения, открыть отдельную карточку — как страница игры в Steam — со всей информацией и действиями именно по этому приложению в одном месте: «я открыл карточку Яндекс.Браузера и хочу иметь все доступные возможности по отношению к нему».

## Область прототипа

- 1-2 категории каталога (например «Браузеры» и «Офис»), не весь список из 71 приложения.
- Данные — из существующего `master.json`, без новых полей в каталоге.
- Никакого пуша/интеграции в основной репозиторий на этом этапе.

## Данные карточки

Источник — существующая модель `AppInfo`/`ClientVersionInfo`/каталожная запись (`name`, `description`, `iconUrl`, `downloadUrl`, `version`, `size`, `category`, `wingetId`/`chocoId`). Два поля выводятся эвристически, т.к. в каталоге их нет:

| Поле карточки | Источник |
|---|---|
| Официальный сайт | Домен, извлечённый из `downloadUrl` (например `download.mozilla.org` для Firefox) |
| «О программе» (тех. детали) | То, что реально есть: версия, размер, категория, ID пакета (winget/choco). Публикатор/лицензия — не показываются, таких данных в каталоге нет |

Баннер карточки — растянутая/размытая существующая `iconUrl` (без новых изображений/скриншотов).

## Макет (вариант B, утверждён пользователем)

Модальное окно (`Window.ShowDialog`), тёмная тема клиента:

```
┌──────────────────────────────────────────────────────┐
│ [иконка] │ Название приложения                         │
│  90x90   │ Категория · статус (установлено/нет) · версия│
│          │ Описание (2-3 строки)                        │
│          │ 🔗 официальный сайт                          │
│          │ Версия: X  Размер: Y МБ                      │
│          ├──────────────────────────────────────────────┤
│          │ [▶ Запустить / Установить]                   │
│          │ [🔄 Переустановить]  (если установлено)       │
│          │ [ℹ О программе]                              │
│          │ [🗑 Удалить]         (если установлено)       │
└──────────────────────────────────────────────────────┘
```

## Триггер открытия

`CatalogTab.xaml`: `TextBlock` с названием сейчас находится ВНУТРИ `CheckBox` (клик по тексту = переключение чекбокса). Меняем разметку строки:
- `TextBlock` с именем выносится из `CheckBox` в отдельный элемент, кликабельный (`MouseLeftButtonUp` → команда открытия карточки), стилизован как ссылка (hover-подсветка).
- `CheckBox` остаётся, но без текста внутри (или с пустым `Content`) — по-прежнему переключает выбор при клике по самому боксу.

## Действия карточки

| Кнопка | Логика |
|---|---|
| ▶ Запустить | Переиспользует `AppLaunchResolver`/существующий `LaunchCommand` из `AppRowViewModel`. При успешном запуске — карточка закрывается (`DialogResult`/`Close()`) |
| Установить | Показывается вместо «Запустить», если приложение не установлено. Переиспользует существующий путь установки (`InstallationService`) для одного приложения |
| 🔄 Переустановить | Новая команда: уже существующий uninstall-путь (адаптируется из `InstalledTab` — `winget uninstall` по ID) + повторный вызов установки |
| 🗑 Удалить | Новая команда: тот же uninstall-путь, что и «Переустановить», без повторной установки |
| ℹ О программе | Показывает то, чего НЕТ в постоянно видимой строке карточки (версия/размер уже видны всегда, см. макет) — категорию, ID пакета (winget/choco), путь установки (`InstallLocation` из реестра, если приложение установлено). Не дублирует уже видимые поля |

Видимость «Переустановить»/«Удалить» зависит от текущего `IsInstalled` строки (тот же признак, что красит `RowBrush` в существующем каталоге).

## Новый ViewModel/View

- `AppCardViewModel` — обёртка вокруг существующего `AppRowViewModel` (переиспользует `LaunchCommand`, `App`, `Availability`, `IsInstalled`), добавляет: `OfficialSiteUrl` (вычисляется из `DownloadUrl`), `InstallCommand`/`ReinstallCommand`/`UninstallCommand`.
- `AppCardWindow.xaml` (+ `.xaml.cs`) — новое модальное окно, открывается из `CatalogViewModel` по команде `OpenCardCommand`, принимает `AppRowViewModel` строки.

## Что НЕ входит в прототип

- Реальные баннеры/скриншоты (используется растянутая иконка).
- Публикатор/лицензия в «О программе» (нет данных в каталоге).
- Массовое распространение на весь каталог (71 приложение) — только демо-категории.
- Интеграция в основной репозиторий/релиз.

## Тестирование прототипа

Ручная проверка через реальный запуск клиента (как при сегодняшней проверке фикса `winget`): открыть карточку установленного и неустановленного приложения, проверить видимость кнопок, проверить закрытие карточки после запуска. Автоматические UI-тесты (FlaUI) — не обязательны для прототипа, добавляются только если решение пойдёт в реальную интеграцию.
