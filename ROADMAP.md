## 3.1.0 — Управление установленным и уведомления
*Фокус: полноценный менеджер установленного + push-уведомления через лаунчер*

- **InstalledTab** — список всех приложений через `winget list`; колонки: название, версия, доступная, источник (✔ winget / ✔ Store / ❓ Unknown)
- **Обновление** — кнопка «Обновить» на каждой строке + массовое «Обновить выбранные» через чекбоксы (`winget upgrade --id --silent`)
- **Удаление** — умный деинсталлятор: winget uninstall → реестр (`UninstallString`) → тихий режим (MSI/NSIS/Inno)
- **Точка восстановления** — WMI `SystemRestore.CreateRestorePoint` перед массовыми операциями
- **HistoryTab + InstallHistoryService** — история установок: поиск, фильтры (успех/ошибка), переустановка одним кликом, хранение до 100 записей
- **Push-уведомления** — ven4admin отправляет сообщение в `Catalog/notifications.json` → git push; все лаунчеры забирают при фоновой проверке и показывают balloon tip из трея
- **Групповое удаление** — чекбоксы + «Удалить выбранные» с подтверждением и точкой восстановления; симметрично массовому обновлению
- **Экспорт / Импорт .winget** — «Экспорт» сохраняет список через `winget export`, «Импорт» восстанавливает через `winget import`; формат совместим с Windows Package Manager
- **Фоновая проверка обновлений приложений** — `UpdateBackgroundService` запускает `winget upgrade` раз в 3 часа; счётчик «⬆ N обновлений» появляется в тексте иконки трея и контекстном меню
- **Crash Reporting + Watchdog** — четыре сценария покрыты:
  - Необработанное исключение → `crash_last.json` со стеком, `SessionId`, `MachineName`
  - Клиент убит извне (Task Manager) → `ProcessKilled` отчёт по коду выхода
  - Клиент завис → `HeartbeatService` (каждые 3 сек) + `WatchdogService` (таймаут 20 сек) → `ProcessFrozen` отчёт
  - Ошибки установки → `InstallFailureService` пишет `failed_installs.json`; `InstallReportWindow` батчит все ошибки в один issue-таблицей
  - Все типы: `CrashReportWindow` / `InstallReportWindow` → «Создать Issue на GitHub» (автозаполнение через API, лейблы `bug`/`crash`/`install-failure`)

## 3.2.0 — Multi-source и офлайн
*Фокус: установка из любого источника, работа без интернета*

- **Chocolatey и Scoop** — поддержка в `InstallationService` наравне с winget и прямыми ссылками
- **SourceOrderService** — настройка приоритета источников: глобально или отдельно для каждой категории
- **OfflineService** — кэширование установщиков (`.exe`, `.msi`) локально; установка без интернета
- **LocalInstallerDialog** — добавление своих установщиков из файловой системы
- **Логирование установок** — `install_YYYY-MM-DD.log` для каждой сессии
- **DebloaterTab** — удаление встроенных приложений Windows (AppX пакеты)
- **DriversTab** — просмотр и удаление установленных драйверов

## 4.0.0 — VenRemote: удалённое управление ПК
*Фокус: управление Ven4Tools с другого ПК через интернет — платная функция (одноразовая покупка)*

### Концепция

Пользователь запускает Ven4Tools → приложение автоматически поднимает встроенный HTTP-сервер →
в интерфейсе появляется публичная ссылка-сессия + PIN → техник подключается со своего ПК
из любой точки и управляет установкой ПО без лишних объяснений.

Для локальной сети (свой ПК + ПК техника на одном Wi-Fi) — бесплатно.  
Для подключения через интернет (relay) — одноразовая покупка, привязка к аккаунту.

---

### Компоненты

#### Встроенный HTTP-сервер `RemoteServerService` ✅ Готово
- `HttpListener` на порту 7777, запускается автоматически при старте Ven4Tools
- PIN-аутентификация (6 цифр, генерируется при каждом запуске)
- Endpoints: `/api/ping`, `/api/apps`, `/api/installed`, `/api/install`, `/api/jobs`
- Показывает `IP:PORT` + `PIN` в панели сайдбара (`pnlRemote`)
- Работает от администратора — URL ACL не нужен

#### VenRemote PC-клиент — вкладка в ProfileTab ✅ Готово
- PRO-карточка «📡 VenRemote» в ProfileTab → кнопка «Открыть →»
- Кнопка «📡 Remote» в сайдбаре (видна только авторизованным пользователям)
- Экран подключения: адрес + порт + PIN (или полный URL для relay-режима)
- После подключения — три вкладки:
  - **Каталог** — поиск и удалённая установка приложений
  - **Установлено** — список ПО на удалённом ПК
  - **Задачи** — статус запущенных установок, авто-обновление каждые 3 сек
- Кнопка «Отключиться», hostname в заголовке после подключения

#### Relay Server — remote.ven4.ru 🔲 К разработке
- Собственный WebSocket-прокси на VPS (~100–150 строк Go/Node)
- Ven4Tools при старте подключается к relay → получает `remote.ven4.ru/s/<id>`
- PC-клиент VenRemote вводит ссылку вместо IP — работает из любой сети
- Не зависит от Cloudflare/ngrok (блокируются в РФ)
- VPS: Selectel / Timeweb Cloud (~300–500 руб/мес), арендуется незадолго до релиза

#### Монетизация 🔲 К разработке
- Одноразовая покупка «VenRemote Pro», привязка к аккаунту через backend
- Проверка: `UserSession.HasFeature("venremote_pro")`
- Без покупки: VenRemote работает только в локальной сети
- С покупкой: доступен relay, сессии через интернет

## 5.0.0 — Windows Image Edition
*Фокус: кастомный дистрибутив Windows с предустановленным Ven4Tools*

> ⚠ Долгосрочная цель. В текущем квартале не реализуется.

- **Кастомный Windows ISO** — образ на базе чистого Windows 10/11, собранный через DISM + `unattend.xml`; Ven4Tools вшит в образ как компонент первого запуска
- **Setup Mode** — специальный режим клиента: запускается полноэкранно при первом входе до появления рабочего стола; пользователь выбирает что установить
- **First-run UI** — упрощённый wizard-интерфейс: выбор категорий / приложений → установка → кнопка «Готово»; после завершения флаг удаляется и открывается обычный рабочий стол
- **Блокировка выхода** — пока установка не завершена, окно не закрывается (перехват Alt+F4, системных кнопок)
- **Автологин + автозапуск** — настраивается через `unattend.xml` / `FirstLogonCommands`; после завершения setup mode убирает себя из автозапуска
- **Сборочный пайплайн** — скрипты для сборки ISO: монтирование WIM, инъекция файлов, генерация `unattend.xml`, перепаковка через Oscdimg
