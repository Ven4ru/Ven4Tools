# Changelog

## [4.3.0] — 2026-07-24

### Установка приложений
- Полоска прогресса установки теперь явно показывает этап: синий цвет — идёт загрузка установщика, зелёный — идёт сама установка. Раньше обе фазы были замешаны в одну шкалу одним цветом, из-за чего было непонятно, на каком этапе процесс.
- Результат установки теперь честно сверяется с фактическим состоянием системы, а не определяется только по коду выхода процесса: если приложение реально появилось в системе — «Установлено» (зелёный), если код выхода успешный, но по факту ничего не изменилось — отдельное предупреждение «Похоже, установлено — не удалось подтвердить» (жёлтый, не путается с настоящей ошибкой), если уже стояла та же версия — «Актуальная версия» вместо ложного «только что поставили».

### Надёжность (Chocolatey)
- Установка через Chocolatey и установка самого Chocolatey теперь имеют собственный таймаут (10 минут) — раньше при зависании стороннего установщика (подтверждено реальным случаем: cpu-z/hwmonitor зависали на 45 минут из-за медленного стороннего сервера) клиент ждал молча без возможности прервать, кроме закрытия всего приложения.
- Устранена возможная порча кириллического текста в выводе Chocolatey из-за отсутствовавшей явной кодировки на части вызовов.

### Прочее
- Устранены находки планового аудита: ложное «100% готово» на общей полоске очереди установки до завершения всех приложений; ложная кратковременная «ошибка» при пропуске источника без SHA256 (не являющемся настоящим сбоем); усиленный резолвинг winget (проверка через Program Files\WindowsApps + Authenticode) портирован в лаунчер, где раньше отсутствовал.

## [4.2.0] — 2026-07-22

### Новая вкладка «Диагностика»
- Новая верхнеуровневая вкладка, заменяет подраздел «Настройки → Диагностика»: информация о системе, управление Turbo Boost, работа с логами — перенесены как есть.
- История перезагрузок и сбоев за последние 7 дней с классификацией: настоящий сбой системы (BSOD), неудачное восстановление из «Быстрого запуска» (Fast Startup) или возможная потеря питания/зависание — с кнопкой отключения «Быстрого запуска», если найдены такие случаи.
- Состояние дисков (SMART/HealthStatus по каждому физическому диску).
- Ошибки установки обновлений Windows за последние 7 дней с кнопкой очистки кэша Windows Update.
- Аппаратные и драйверные события (WHEA, сбои видеодрайвера) — только счётчики и сырые записи, без автоматического диагноза.
- Кнопка «Скопировать полный отчёт» — текстовый дамп всех разделов для поддержки.
- Все проверки — только по кнопке «Запустить диагностику», ничего не работает в фоне; вкладка не делает сетевых запросов.

### Каталог
- Поиск по каталогу: если запрос не находит совпадений локально, теперь распознаёт и название категории («видео», «антивирус», «мессенджеры» и другие), а не только имя пакета — подсказки из Winget подбираются по тегу манифеста, а не только по буквальному имени.

### Исправления
- Автофикс «Очистить кэш Windows Update» мог тихо ничего не очистить (если службы не остановились) и всё равно показать «Успех» — теперь настоящий сбой всплывает с подсказкой запустить от имени администратора.

## [4.1.9] — 2026-07-20 (лаунчер — v3.1.4)

### Лаунчер
- Выбор источника загрузки в настройках: «Авто» (по умолчанию), GitHub, CDN (домен), CDN (прямой IP) или зеркало на хостинге. Выбранный источник ставится первым в цепочке, остальные остаются резервом — фоллбэк не отключается.
- Автоматический фоллбэк расширен с двух источников (CDN → GitHub) до четырёх: CDN (домен) → CDN (прямой IP в обход DNS) → зеркало на хостинге (независимый провайдер) → GitHub. Актуально для пользователей в России, у которых github.com режется по SNI, а домен CDN теоретически может быть заблокирован по DNS отдельно от IP.
- Обход блокировки домена CDN по DNS: если имя `cdn.ven4tools.ru` не резолвится, `version.json` перезапрашивается по прямому IP (из подписанного манифеста) без ослабления проверки TLS-сертификата — SNI и валидация сертификата остаются штатными.
- Исправлено: проверка наличия обновления самого лаунчера шла только через GitHub API. Теперь основной источник обнаружения версии — подписанный `version.json` CDN (симметрично клиенту), GitHub — резерв. Раньше при недоступном GitHub лаунчер не узнавал о вышедшем обновлении, даже когда CDN был доступен.
- Зеркало на хостинге доверяется только для пути `/releases/` — остальной сайт (включая API) не считается доверенным источником загрузки. SHA256 обязателен для каждого источника, включая зеркало и прямой IP.

### Каталог
- Карточка приложения: клик по названию в каталоге открывает окно с описанием, ссылкой на официальный сайт, версией/размером/ID пакета и кнопками «Запустить»/«Установить», «Переустановить», «Удалить» (видимость зависит от того, установлено ли приложение).
- Карточка приложения теперь отражает статус строки каталога в реальном времени — раньше замирала на состоянии на момент открытия и не подхватывала фоновое обновление (доустановка, изменение версии).
- Чекбокс выбора приложения увеличен до 20×20 — раньше кликабельная область по факту оставалась заметно меньше отображаемого квадратика (реальная зона хит-теста была ограничена глифом, не заданным размером).

### Интерфейс
- Все окна клиента и лаунчера теперь используют тёмный заголовок (native title bar) при тёмной теме Windows — раньше строка заголовка оставалась светлой даже в тёмной теме.
- Кнопка «Удалить» в списке установленных приложений: контур по умолчанию и красная заливка только при наведении вместо сплошной красной заливки в каждой строке (при большом списке приложений сплошная заливка выглядела как стена предупреждений). Кнопка массового удаления не затронута.

## [4.1.8] — 2026-07-17

### Каталог
- `winget` теперь резолвится напрямую из установленного пакета App Installer (`Program Files\WindowsApps`), а не только через персональный alias — на части машин ACL этого alias-каталога оказывался шире узкого предположения "только SYSTEM/Administrators/TrustedInstaller" без реальной компрометации, и весь каталог без прямых ссылок на установщик ложно показывался недоступным.
- Authenticode-проверка (издатель Microsoft) вынесена в переиспользуемый сервис вместо дублирования между клиентом и лаунчером.

## [4.0.4] — 2026-07-13 (лаунчер — v3.1.1)

### Безопасность
- Клиент: установка локального установщика без валидного SHA256 теперь отклоняется (fail-closed) — раньше при пустом или повреждённом хеше проверка целостности пропускалась целиком.

- Лаунчер: защита от установки или удаления клиента в известные пользовательские папки (Документы, Рабочий стол, Program Files, Загрузки) целиком — раньше проверялось только пересечение с папкой данных Ven4Tools.

- Клиент: убрана рудиментарная функция — диалог добавления приложения (`AddAppDialog`) нигде не был доступен из интерфейса.

- **Вкладка "Установленные"** — полноценный менеджер установленных приложений
  - Список всех приложений через `winget list` с колонками: название, версия, доступная версия, источник
  - Фильтры: Все / Нужно обновить / Неизвестный источник
  - Поиск по названию и Winget ID
  - Кнопки "Обновить" и "Удалить" на каждой строке
  - Массовое обновление: выбор через чекбоксы + "Обновить выбранные"
  - Бейджи источника: ✔ winget / ✔ Store / ❓ Неизвестный
  - Статистика: Всего / Обновлений / Неизвестных
- **Умный деинсталлятор** (три уровня):
  1. `winget uninstall --id` для пакетов с известным источником
  2. Поиск `UninstallString` в реестре (`HKLM/HKCU\...\Uninstall\*`) по `DisplayName`
  3. Тихий режим: msiexec `/quiet /norestart`, NSIS `/S`, Inno Setup `/SILENT` — автоматически

### Исправлено
- Парсер `winget list`: прогресс-бар winget склеивался с заголовком в одну строку — теперь префикс корректно обрезается
- После успешного обновления приложение сразу исчезает из фильтра "Нужно обновить" (ранее оставалось до следующего обновления списка)
- Свойство `Available` теперь уведомляет биндинг `HasUpdate` через `INotifyPropertyChanged`
- Null guard в `ApplyFilter()` устраняет `NullReferenceException` при инициализации XAML

- **Флаг `ruBlocked` для приложений** — каталог поддерживает метку для приложений, загрузка которых заблокирована в РФ (CDN издателя недоступен)
  - Приложения с флагом помечаются бейджем **⚠ РФ** в каталоге
  - Кнопка 🌍 в строке поиска — переключатель «показывать / скрывать» такие приложения
  - Настройка сохраняется в профиле пользователя и переживает перезапуск

### Известные ограничения
- Steam-игры без ARP-записи в реестре (например, некоторые тайтлы) не удаляются без активной сессии Steam

### Служебное
- Удалены морально устаревшие релизы ниже v2.3.2 (v2.2.0–v2.3.1) — разработка и улучшение архитектуры идут дальше

## [3.0.1] — 2026-05

### Обновления

#### Лаунчер (Ven4Tools.Launcher)
- **Авто-проверка при запуске** — лаунчер автоматически проверяет наличие компонентов и обновлений при старте, без ручного нажатия
- **Кнопка «Найти Ven4Tools на диске»** — автоматический поиск установленного клиента в Program Files, Документах и Загрузках; работает на системах с русским и английским интерфейсом
- **Кнопка «История изменений»** — заменяет стрелочный переключатель; открывает боковую панель с отформатированными release notes
- **Кнопка отмены загрузки** — позволяет прервать скачивание версии клиента в любой момент
- **Скрытые кнопки действий** — «Обновить лаунчер» и «Установить компоненты» теперь скрыты и появляются только когда действие реально доступно; убраны залоченные кнопки
- **Выбор версии** — в комбо-боксе версий теперь отображается дата выпуска и размер файла

#### Клиент (Ven4Tools)
- **Вкладка Office — разделение кнопок** — «Скачать и установить» разбита на отдельные «📥 Скачать» и «🚀 Установить»; скачивание и запуск установщика теперь независимые шаги
- **Вкладка Office — статус региона** — постоянный блок в интерфейсе показывает текущий регион Windows (GeoID) и Office CountryCode в реальном времени; обновляется в момент переключения при установке
- **Вкладка Office — отмена установки** — кнопка «Отмена» теперь активна на протяжении всей фазы установки, а не только при скачивании

### Исправления

#### Лаунчер
- `UpdateBackgroundService` — `async void` в колбэке таймера заменён на безопасный паттерн `_ = CheckAllAsync()`; ранее необработанное исключение крашило процесс
- `GitHubService` — добавлен `using` во всех четырёх точках вызова; утечка HTTP-соединений устранена
- `DownloadVersionAsync` — состояние кнопки отмены не восстанавливалось в блоке `finally`
- `CheckComponentsInteractiveAsync` — условие проверки обновления лаунчера сравнивало `IsEnabled` вместо `Visibility`; кнопка никогда не считалась видимой
- `RestartAsAdmin` — сервисы и `NotifyIcon` не диспозились перед вызовом `Shutdown()`
- `BtnLaunchApp_Click` — убран `Verb = "runas"`, добавлен `MessageBox` при исключении
- `InstallWingetAsync` — `shutdown /r` запускался без `UseShellExecute = true`; процесс не стартовал
- `UpdateService` — отсутствовал null-guard на `DownloadUrl` перед вызовом `GetAsync()`
- `CmbVersions_SelectionChanged` — release notes отображались даже когда боковая панель была закрыта
- Удалён мёртвый метод `CheckDotNetInstalled()`, который компилировался но никогда не вызывался

#### Клиент
- `InstallationService` — инъекция команд через `cmd.exe`; исправлено прямым вызовом `winget.exe` с экранированием `--id "{appId}"`
- `ZapretService` — мутация `HttpClient.Timeout` в конструкторе после первого запроса; исправлено статическим инициализатором
- `AvailabilityChecker` — `Dictionary<>` заменён на `ConcurrentDictionary<>`; устранено состояние гонки при параллельных проверках
- `InstallationService` — `Process` без `using`; утечка ресурсов при установке
- `InstalledTab` — `ExitCode` читался после истечения таймаута `WaitForExit()`; теперь корректно обрабатывается через флаг `exited`
- `HashHelper` — пропуск проверки SHA-256 происходил молча; добавлен `Debug.WriteLine`
- `AppManager` — непоследовательное использование `lock(lockObj)` на коллекциях `alternatives` и `hiddenApps`; возможны были data race
- `CatalogTab` — исключения из `FetchVersionsPhase2Async` терялись в `ContinueWith`; лямбда переписана на `async`/`await`
- `CatalogTab` — `InstallationService` и `AvailabilityChecker` не диспозились в обработчике `Unloaded`
- `ActivationTab` — `curl.exe` заменён на `iwr -UseB` (PowerShell-native); устраняет зависимость от наличия `curl` в PATH
- **Вкладка Office — прогресс «0/7 МБ»** — целочисленное деление `total / 1_048_576` давало 0 до скачивания первого МБ; исправлено на `(double)total / 1_048_576`
- **Вкладка Office — полоса не сбрасывалась при отмене** — `Task.Delay(3000)` в `finally` создавал гонку с новым запуском; прогресс теперь сбрасывается немедленно в `catch`
- **Вкладка Office — кнопки зависали после закрытия установщика** — фаза установки не имела `CancellationToken`; добавлен CTS, кнопки разблокируются мгновенно по «Отмена»
- **Вкладка Office — регион не менялся** — `SetCountryCode` менял только Office-специфичный ключ `ExperimentConfigs\Ecs`; теперь также изменяется `Control Panel\International\Geo\Name` и `Nation` (GeoID 244)
- **Вкладка Office — зависание после ошибки установщика** — при ненулевом exit code bootstrapper код ждал C2R-процесс 3 минуты; теперь сразу выходит с диагностическим сообщением
